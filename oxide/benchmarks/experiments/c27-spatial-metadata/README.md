# C27 spatial-metadata evidence

C27 prepares conservative raster/effect/glyph/mesh bounds, nested resolved clips, matching scope endpoints, and top-level paint spans when an immutable chunk is created. Large property-free snapshots resolve their instance transforms and build one compact ordered x-sweep index. Dynamic-property snapshots retain C26's allocation-free construction path. Metal uses the two-level snapshot/chunk query only for small damage; full damage bypasses querying and unchanged static snapshots reuse a plan after validating its unique prepared keys.

The target workload contains 10,000 alternating glyph and image-mesh instances. Small two-pixel damage must visit and draw one instance/command, revisit zero source vertices, copy/upload zero warm geometry bytes, shade four pixels, and preserve the full-frame readback. Full damage must visit zero query entries, replay 10,000 draws linearly, reuse one validated static plan, and protect frame/encode/GPU tails. The permanent engine rows are `cpu.architecture.spatial_metadata.glyph_mesh_10000` and `gpu.architecture.spatial_metadata.{small_damage,full_damage}_glyph_mesh_10000`; the public contracts are `cpu.authoring.retained_snapshot.spatial_query_10000` and `gpu.authoring.retained_snapshot.spatial_damage_10000`.

The exact parent/candidate probe runs at user-interactive QoS with one complete full-frame warmup, 128 measured-path warmups, 250 observations per fresh process, 15 balanced pairs, completed-command-buffer GPU timing, and final BGRA8 readback hashing. Separate paired decisions cover small-damage frame/encode/GPU distributions and full-damage frame/encode/GPU distributions. The locally ignored `raw/` directory retains the instrumentation patch, exact tree/binary identities, every sample, order, counter result, and decision. Aggregate workspace and physical-device baseline promotion remains deferred to C62.

Three alternatives were rejected before final measurement. Eager resolved-instance/index construction more than doubled the C26 300-node animation CPU row. Lazy `OnceLock` construction still regressed that row and added synchronization/layout overhead to every dynamic snapshot. Rebuilding the identical 10,000-entry full-frame Metal plan on every frame left CPU submission outside the strict tail envelope; retaining the plan only for the same property-free snapshot and viewport removed that repeated work while unique prepared-key validation preserves resource and target correctness.

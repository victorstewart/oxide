# C39 WebGPU compact nine-slice evidence

C39 replaces the WebGPU CPU expansion of each `NineSlice` into as many as nine generic image instances with one 44-byte destination/image-size/insets/alpha instance over a persistent 36-vertex, 54-index cell grid. Adjacent instances share one ordered draw only while texture, texture format, target, and clip remain compatible. Per-cell vertices deliberately remain distinct: a zero-width or zero-height source cell must retain the existing full-image source fallback without introducing UV interpolation across a neighboring cell.

The isolated real-Chrome matrix runs 1, 64, 512, and 1,024 nine-slices. Every row rotates through normal fractional insets, destination rectangles smaller than their fixed edges, asymmetric fractional insets, and clamped negative/oversized insets. Parent and candidate packages use the same host-only adapter, native arm64 Chrome/Metal, queue-completed CPU samples, and in-app WebGPU timestamp queries.

The candidate uploads 44 bytes per logical nine-slice instead of up to 324 bytes. In the pathological matrix this reduces uploads by 86.4% for one primitive and 67.4% for the 64/512/1,024 rows, where the parent drops invalid cells before upload. It preserves one draw and one texture bind in every row. The three-pair pre-freeze population showed CPU p50 changes of +7.0%, -13.1%, -25.2%, and -38.8% at 1/64/512/1,024 primitives; CPU p95 changed by -8.2%, -10.8%, -28.1%, and -33.5%. Direct GPU p50 changed by +8.3%, -13.8%, -3.6%, and +7.7%, or roughly +0.0021, -0.0065, -0.0022, and +0.0050 ms. The 1,024 row's small GPU regression is the measured cost of retaining all 18 fixed-grid triangles when the parent can omit collapsed pathological cells; the 0.239 ms CPU median reduction is substantially larger than that 0.005 ms GPU increase. C39 is accepted for the whole-frame CPU/upload win with exact pixels, unchanged draw/bind counts, and explicit disclosure that it is not a universal GPU win. The final exact-tree raw population is the acceptance source of truth.

The deterministic 8×8 capture covers two RGBA textures, fractional and small destinations, negative and oversized insets, zero-width source intervals, four opacity levels, clipping, filtering, and seam boundaries. Parent/candidate pixels match exactly at DPR 1, 2, and 3. The 1,200×800 prepared snapshot also matches exactly, proving the compact grid through prepared direct/bundle ownership plus dynamic transform/opacity property records. Aggregate committed browser/workspace baseline promotion remains deferred to C62.

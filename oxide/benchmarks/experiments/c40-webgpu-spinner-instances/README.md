# C40 WebGPU procedural-spinner evidence

C40 replaces the WebGPU lowering of each `Spinner` from twelve CPU `sin`/`cos` pairs and twelve analytic RRect records with one 20-byte center/atom/alpha/color instance. One prebuilt spinner pipeline generates twelve six-vertex atom quads from immutable WGSL directions and reuses the analytic RRect fragment shader. Adjacent compatible spinners retain one ordered draw. The existing viewport/property uniform carries animation phase, transform, and opacity, and its 48-byte write occurs only when a direct or prepared frame contains spinner work.

The isolated native-arm64 Chrome/Metal matrix runs 1, 64, 512, and 1,024 spinners with six CPU samples of 24 queue-completed frames and 144 in-app WebGPU timestamp samples per row. Calibration reduced logical instance uploads from 432 to 20 bytes per spinner plus one shared 48-byte time/property record: 432 to 68 bytes for one spinner, 27,648 to 1,328 bytes for 64, 221,184 to 10,288 bytes for 512, and 442,368 to 20,528 bytes for 1,024. Triangle and draw counts remain 24 per spinner and one compatible draw. CPU p50 changed by +10.4%, -27.5%, -68.2%, and -77.8%; direct GPU p50 changed by +2.9%, -40.3%, -37.7%, and -61.5%. The one-spinner deltas are sub-0.03 ms noise and are disclosed rather than presented as a universal win.

The displayed-animation probe renders 64 spinners for 600 real RAF callbacks. Its calibration pair used the CPU-expanded phased RRect reference against the compact procedural path. CPU submit p50 fell from 0.190 to 0.120 ms and direct GPU p50 from 0.0574 to 0.0434 ms. RAF p95 moved from 8.770 to 8.985 ms, RAF p99 from 9.205 to 9.330 ms, and neither path hitched; both drained the timestamp queue to zero with 600 CPU, RAF, and GPU samples. This is a renderer-path pacing diagnostic rather than a claim about end-to-end input latency.

The 8×8 phased capture varies radius and opacity, retains clipping, and compares the candidate against the old CPU-expanded RRect construction at four animation phases. DPR 1 and 2 comparisons are exact. At DPR 3, 280 of 589,824 pixels differ only on seven copies of one antialiased atom; maximum channel error is 6 and MSE is 0.001. This 0.047% boundary-only delta comes from CPU-versus-GPU floating-point evaluation order and is accepted because restoring per-dot CPU geometry would erase the measured objective. Prepared direct/bundle paths retain the same 20-byte records and use the existing dynamic property uniform for transforms, opacity, and phase; focused tests freeze that ownership.

C40 is accepted for the decisive dense CPU, upload, and GPU wins, unchanged logical geometry/draw count, bounded animation pacing, and visually equivalent phased output. The final exact-tree balanced population is the acceptance source of truth. Aggregate committed browser/workspace baseline promotion remains deferred to C62.

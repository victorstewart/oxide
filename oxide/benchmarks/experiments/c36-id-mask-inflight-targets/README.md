# C36 Metal ID-mask in-flight target evidence

C36 removes the snapshot/offscreen renderer's eight-entry array of large ID-mask target sets. Cache entries now own immutable static generations, submitted frame slots retain only small generation serial/byte records, and the snapshot readback path owns one lazy target set. A completed same-size transient target is recycled; a dimension mismatch releases it before allocation. Cache eviction may drop its CPU handle while Metal still owns an active command buffer, but the evicted textures cannot be returned for rewriting until every referencing completion bit clears.

The affected-path proof compiles the unchanged C35 parent and candidate with `oxide-renderer-metal/snapshot-tests`, disables the field cache, warms all eight offscreen slots, advances to an explicit allocated-memory sample, and exercises steady 512, 1024, 2048, and repeating 512/1024/2048 churn. Parent-to-candidate steady allocated ownership is exactly 38,273,024 to 4,784,128 bytes at 512, 152,043,520 to 19,005,440 at 1024, and 608,174,080 to 76,021,760 at 2048: one target set instead of eight, or 87.5% less. Churn peak falls from 280,428,544 to 76,021,760 bytes, or 72.9%. Across the eight-frame calibration population, steady texture creation falls from 32 to zero after warmup; churn creates 32 textures on both sides because every adjacent dimension differs. All cases report zero backpressure and skipped submissions.

The calibration's parent/candidate completed-command-buffer GPU medians were 0.551000/0.500625 ms at 512, 2.136500/1.300000 ms at 1024, 5.442042/3.569458 ms at 2048, and 1.287458/1.259750 ms under size churn. These timings establish that removing residency and allocation pressure does not buy memory by adding GPU work; the frozen-tree paired report in the ignored `raw/` directory is the acceptance authority.

Because the old target array was populated only by `snapshot-tests`, C36 also runs a separate non-snapshot production guardrail. In the five-pair calibration with forty samples per side per workload, the worst candidate GPU movement was the 2048 p99/peak at +2.475%, and the worst frame movement was the 2048 p99/peak at +4.055%; both remain inside the fixed 5% no-material-regression threshold. Parent production memory reports zero for cache-disabled command-buffer-only targets because C35 could not see that ownership. The candidate reports one active generation through the new generation-aware accounting; this corrects telemetry rather than retaining another target.

Reference proof hashes the final BGRA target for every workload. Parent and candidate hashes are identical for 512, 1024, 2048, and the churn endpoint. Focused Metal tests additionally force a cache generation to remain busy, require a second allocation instead of overwrite, verify exact double-generation bytes and zero backpressure, compare complete output bytes, release the busy slot, and require zero-create compatible reuse on the next frame.

The renderer exposes per-frame target creates, unique in-flight generations and allocated bytes, cache-plus-in-flight storage, lifetime peak storage, and synchronization-blocked reuse. The architecture matrix persists these counters for the existing ID-mask rows. The ignored `raw/` directory retains the probe, alternating populations, exact source and binary identities, memory/create/backpressure arrays, pixel hashes, and decision reports. Aggregate workspace and physical-device baseline promotion remains deferred to C62.

# C32 Metal ID-mask field-cache evidence

C32 retains complete immutable Metal ID-mask raster and jump-flood fields when geometry and projection are unchanged. The exact key includes dimensions, mask scale, aggregate vertex/chunk content revisions and ranges, and every raster projection input. Style, color, glow, polish, mode, opacity, and viewport placement remain final-compositor inputs and do not invalidate fields.

Cache hits encode only the final compositor. Misses retain the existing raster, seed, and logarithmic JFA sequence. At 512 square this changes warm static/style/viewport work from twelve render passes to one; projection and content changes remain twelve-pass misses, while the two-map case produces two independent cache hits and two compositor passes in one command buffer.

The cache holds at most four immutable field sets under an adaptive 64–512 MiB Metal allocated-byte budget. Current-command-buffer entries cannot be recycled; later command buffers are ordered by the renderer's single tracked Metal queue, and committed command buffers retain resources through completion. Compatible evicted targets are rewritten without allocation. Production frame slots do not retain historical targets; snapshot builds keep the latest slot handles only for exact readback. Explicit purge, iOS memory pressure, and device loss release cache ownership.

The controlled static probe used the unchanged C31 parent, identical instrumentation patch `b453873d580a7c35b1fa0ec3cf8c12535e23881bdf1b8cf065b1d8d3f1f45dea`, twelve symmetric projection-changing GPU clock warmups on a disposable renderer, a fresh measurement renderer, eight cache/ring warmups, 140 measured frames per process, and fifteen fixed-seed balanced pairs. The 2,100 completed-command-buffer samples per side measured GPU p50/p95/p99/peak of 1.536042/1.571637/1.641635/2.137958 ms on the parent and 0.065625/0.070750/0.072459/0.174417 ms on the candidate. Paired median speedup was 95.653% with 95% CI 95.494%..95.746% and 15/15 pair wins.

Across that population, per-process measured passes fell from 1,680 to 140, steady ID-mask target ownership fell from 139,198,464 to 17,399,808 bytes, and measured resource creation stayed zero after warmup on both sides. Frame p50/p95/p99 improved 52.077%/54.799%/54.651%; Metal encode p50/p95/p99 improved 93.158%/86.052%/79.375%.

Metal validation runtime tests cover final-only hits, projection/revision/chunk/scale/dimension misses, same-command-buffer two-map safety, hard-budget eviction including a deferred zero-budget boundary, explicit purge, and iOS memory-warning routing. Snapshot proof compares a compositor-only hit with a fresh full field build and requires exact final BGRA bytes plus exact R8 masks and RGBA32F city/seam fields. The permanent six-scenario smoke report records static/style/viewport as one-pass hits, projection/content as twelve-pass misses, and the two-map workload as two one-pass hits.

The locally ignored `raw/` directory retains both harness-refinement populations, instrumentation patches, exact source/binary identities, balanced order, every GPU/CPU sample, workload counters, and deterministic decision reports. Aggregate workspace and physical-device baseline promotion remains deferred to C62.

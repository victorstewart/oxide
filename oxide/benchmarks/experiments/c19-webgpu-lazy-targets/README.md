# C19 WebGPU lazy-target evidence

C19 sets the canvas backing dimensions before WebGPU surface construction, removes the browser's transient 300×150 renderer allocation, and makes full-size scene, backdrop-scratch, and Scene3D-depth targets optional. Direct UI owns none of those targets. Backdrop creates scene plus scratch, Scene3D creates depth, and physical resize drops incompatible handles for completion-safe wgpu reclamation. The viewport uniform is written at construction and resize instead of every submit.

The performance hypothesis is that eager construction and every resize perform three unnecessary full-size texture creations for direct UI, retain 12 bytes per pixel of auxiliary color/depth storage, and upload an unchanged viewport on every frame. The affected stages are WebGPU construction, surface resize, resource residency, first backdrop/Scene3D use, and direct submission. Expected movement is construction texture/bind-group count from 7/7 toward the base renderer's 1/3, direct auxiliary bytes from 12 bytes per pixel to zero, backdrop bytes from 12 to 8 bytes per pixel, Scene3D bytes from 12 to 4 bytes per pixel, resize target creation from three to zero, and one fewer 16-byte viewport upload on ordinary frames. The risk is a first-feature hitch, stale-size target reuse, invalid in-flight destruction, or changed backdrop/Scene3D pixels.

The first 15-pair pilot confirmed the resource reduction and a 42.5% resize median win, but it also showed that unannounced lazy first use moved synchronous allocation into backdrop and Scene3D submission. That design was not accepted as final. The retained API lets an app prewarm only backdrop's scene/scratch pair or Scene3D's depth target outside a latency-sensitive frame. The C19 adapter reports prewarm cost and prewarm-plus-first-frame ready time; the parent receives an identical benchmark-only no-op method because its resize has already created all three targets.

Every fresh Chrome process records one separate warmup and 50 raw repetitions for direct resize/first/ready/completion/GPU, backdrop prewarm/first/ready/completion/GPU, and Scene3D resize/prewarm/first/ready/completion/GPU. Prototype interception counts real `GPUDevice.createTexture` and `createBindGroup` calls only around app construction, then restores the methods before measured frames. The locally ignored `raw/` directory retains the exact instrumentation patch, paired plans, fixed order, process outputs, hashes, startup distributions, and visual captures. Official `latest.*` baseline promotion remains deferred to C62.

# C37 WebGPU analytic RRect evidence

C37 replaces WebGPU's per-frame rounded-rectangle tessellation and trigonometry with one persistent 36-byte rect/radii/packed-color instance per command. WGSL expands six immutable unit corners and evaluates the proven corner-selected signed-distance function. Adjacent instances share one ordered draw only when target and clip remain compatible. Existing borders remain ordered outer/inner RRect instances, and retained dynamic transforms/opacity continue through the prepared property record selected for the chunk instance.

The isolated real-Chrome matrix runs 1, 64, and 1,024 ordinary RRects plus 64 finite pathological radii at DPR 1, 2, and 3. Parent and candidate builds use the same host-only adapter, native arm64 Chrome/Metal, queue-completed samples, and in-app WebGPU timestamp queries. Normal RRects fall from 36 to two triangles and from 956 to 36 uploaded bytes each after C16's already-compacted parent geometry: 94.4% fewer triangles and 96.2% fewer bytes. A 1,024-RRect frame falls from 978,944 to 36,864 uploaded geometry bytes while retaining one draw.

The final campaign used three order-balanced parent/candidate launches at each DPR, with six CPU samples and 144 queue-completed GPU timestamps per row in each launch. The median process CPU p50 fell by 52.1/53.3/52.0% at 64 ordinary RRects, 84.0/83.3/85.1% at 1,024, and 51.6/50.1/53.2% for pathological radii across DPR 1/2/3. Even the noise-sensitive singleton improved by 10.9/15.4/6.7%. Direct GPU time did not provide a universal win: the 64-RRect p50 improved by 10.6/3.6/1.1%, while the 1,024 row changed by +1.5/+10.1/+32.6%. The worst absolute candidate result was 0.095832 ms versus 0.072249 ms at DPR3, a 0.023583 ms increase, while CPU lowering fell from 3.548 to 0.528 ms. C37 is accepted for the decisive CPU/upload/triangle reduction and sub-0.1 ms GPU total; it does not claim a universal GPU win.

The deterministic 8×8 capture covers zero, negative, fractional, asymmetric, half-extent, and oversized radii plus narrow destinations. At DPR 1/2/3, parent/candidate differences were 6,648/13,904/21,744 pixels. `scripts/check_c37_rrect_pixels.mjs` classifies the physical pixel footprint against the analytic boundary: every differing pixel was within 0.793 physical pixels and every pixel outside the predeclared one-pixel AA boundary tolerance matched exactly. The paired exact-tree raw directories retain reports and captures, and the candidate raw directory retains classifier output. Aggregate committed browser/workspace baseline promotion remains deferred to C62.

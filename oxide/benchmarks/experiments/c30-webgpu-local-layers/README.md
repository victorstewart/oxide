# C30 WebGPU local-layer evidence

C30 replaces one full-canvas WebGPU texture per cached layer with a persistent texture covering only the layer's transformed bounds plus effect outset. Bounds snap outward to the canvas physical-pixel grid so fractional origins retain the parent's raster phase. Every target owns a reusable local viewport; nested targets inherit the ancestor matrix/translation, clips are conservatively transformed into local scissors, backdrop copies cover only the visible local span, and final composites use normalized UVs.

The prepared key covers stable layer and chunk identity, structural/geometry/resource/dynamic generations, bounds, diagonal scale, opacity, target scale/format/device generation, effect outset, physical pixel phase, and exact resource dependencies. Clean matches skip the chunk body immediately. Dirty or missing content clears and redraws once into a dimension-compatible texture; image update/release, prepared purge, scale, and device generation invalidate dependent keys without admitting stale pixels. Invalid, empty, sheared/rotated, dynamically clipped, unbounded-effect, or device-limit-exceeding retained layers take the exact fallback.

The permanent workload contains 100 retained 72×40-point cards at DPR2. It expects 4,608,000 texture bytes at both physical 1920×1080 and 3840×2160, versus parent full-canvas residency of 829,440,000 and 3,317,760,000 bytes. Clean frames require 100 hits, 500 skipped commands, zero body traversal, and one final pass. One-dirty frames require 99 hits, one miss, five traversed body commands, one local layer pass, three total passes, and zero recurring texture creation.

The 512×512 DPR2 edge oracle combines a partly offscreen fractional retained layer, internal negative-origin clip, nested local layer, backdrop sampling, and clean replay. The initially unsnapped/non-inherited design was rejected after it changed 64,410 pixels. The retained design produces the same PNG SHA-256 as the unchanged parent with zero pixel differences, zero maximum channel error, and zero MSE.

The first pixel-correct implementation still recomputed every layer frame/key twice and used two hash passes per clean snapshot. Its exact 1080p clean population was rejected: CPU regressed 153.157% with a paired 95% interval of -158.261%..-147.466% and 0/15 pair wins, while direct GPU median regressed 25.775%. The retained implementation instead caches the validated immutable layer plan by snapshot identity and clears that plan with key-space-changing lifecycle events; clean frames now perform only cache validation and composite emission.

The first retained-plan population proved the CPU fix but also exposed asymmetric GPU clock conditioning: the parent's 829 MB population clear consistently entered the fast clock state, while the 4.6 MB candidate alternated between roughly 0.056 and 0.073 ms. That methodology was rejected after CPU improved 6.121% with 15/15 wins but the direct GPU paired median appeared 9.807% slower. Final timing applies the same fixed twelve-frame, 64-draw, 256-point-square clock warmup to both binaries after population and before the four recorded layer warmups; none of that work enters measured samples.

The symmetric-warmup 1×150-frame topology then improved CPU 6.388% with 15/15 wins and improved the paired GPU median 0.755%, but failed only the GPU peak gate: every >0.12 ms outlier on both sides occurred at terminal sample 149 before the long-burst drain, including one 0.230416 ms candidate peak. That collection topology was rejected rather than filtering samples. Final processes retain all 150 timestamps as ten 15-frame blocks with a completion drain between blocks; a direct parent/candidate probe reduced peaks to 0.127986/0.095365 ms while preserving the same scene, warmup, total frame count, and fresh-process contract.

The formal one-dirty 10×15 population still placed its only decision-breaking candidate outlier at global sample 149: GPU median/p95/p99 improved 3.464%/3.478%/4.211%, but the 0.174399 ms terminal frame exceeded the 0.140282 ms parent peak. Final collection therefore appends one declared-unmeasured same-work postroll frame and removes only that frame's exact timestamp frame ID. This preserves all 150 measured timestamps while ensuring no measured frame doubles as the process/drain terminal marker.

The locally ignored `raw/` directory retains the instrumentation patch, exact parent/candidate tree and wasm identities, balanced pair order, every CPU and GPU sample, health counters, workload counters, exact captures, aggregate reports, and final decision. Official browser baseline promotion remains deferred to C62.

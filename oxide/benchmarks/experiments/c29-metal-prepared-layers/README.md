# C29 Metal prepared-layer evidence

C29 gives eligible retained snapshot layers complete stable keys covering layer identity, chunk/content/nested/dynamic generations, exact resource dependencies, bounds, diagonal scale, opacity, target scale/format/sample count, device generation, and bounded effect outset. Translation remains dynamic. A clean hit composites the retained texture without consulting the prepared body, scanning or copying commands, copying geometry, uploading buffers, creating a texture, or opening an offscreen pass.

A missing or dirty layer prepares its immutable body if necessary, renders it once into a compatible private texture, then composites once. Repeated occurrences of one stable key share that refresh. Resource update/release and prepared-cache purge invalidate dependent layer keys without discarding a compatible texture. Shear, rotation, instance clips, effects, internal layers, spinners, mixed opaque/translucent RRect precision, and conflicting duplicate IDs retain the exact parent flat path.

Prepared layers preserve the parent C05 layer blend/composite contract. Translucent RRects and image/glyph/image-mesh/Solid bodies use the main layer format. Effective-opacity-one RRects use RGBA32Float because main-format and RGBA16Float trials changed exact antialiased edge bytes. The universal RGBA32Float policy was also rejected because it changed the parent quantization of overlapping translucent RRects.

The permanent rows are `gpu.architecture.prepared_layers.clean_100x100`, `gpu.architecture.prepared_layers.one_dirty_100x100`, and `gpu.authoring.retained_snapshot.prepared_layers_clean_100x100`. The clean workload contains 100 retained layers with 100 RRects each and requires 100 hits, one pass, 100 composites, and zero body/offscreen/upload/texture/preparation work. The dirty workload requires 99 hits, one miss, one offscreen replay from the existing prepared body, two passes, 101 total draws, and zero warm copy/upload/preparation/texture creation. Separate generation/resource/scale tests cover missing-body invalidation; the timed dirty row isolates C29 refresh ownership instead of repeating C24 buffer churn.

The locally ignored `raw/` directory retains the byte-identical instrumentation patch, exact parent/candidate tree and binary identities, balanced pair order, every frame/encode/completed-command-buffer GPU sample, counters, output fingerprints, and final decisions. Aggregate workspace and physical-device baseline promotion remains deferred to C62.

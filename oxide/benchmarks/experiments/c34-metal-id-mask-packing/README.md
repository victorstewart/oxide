# C34 Metal ID-mask field-packing evidence

C34 stores city-seed XY in `.xy` and seam-seed XY in `.zw` of two ping-ponged RGBA16Uint fields whenever both maximum coordinates are below the reserved `0xFFFF` invalid value. The final compositor resolves city and neighborhood IDs from the authoritative R8 masks at the selected coordinates. Four RGBA32Float city/seam fields and their prebuilt shaders remain the exact coordinate-overflow fallback; ordinary Metal texture limits select the packed path.

The packed fields use 16 logical bytes per pixel across both ping-pong textures versus 64 for the four wide fields, an exact 4x reduction. Each JFA candidate now fetches one eight-byte city/seam coordinate record instead of two sixteen-byte records. At final composition, direct city/neighborhood IDs bypass recovery, while fallback and seam pixels read only the needed R8 ID at the chosen seed.

The controlled 512-square forced-miss proof used unchanged C33 parent `55b8287c822bf97188f01b74fedcdbd90e7f9c4f`, identical instrumentation patch `1f36f16a9f63b2c96d86cbb07df1732e5cb8689e41e37cc06cfdf6de374e8cc1`, twelve symmetric forced-miss clock warmups on a disposable renderer, a fresh measurement renderer, eight forced-miss cache/ring warmups, 140 measured frames per process, and fifteen fixed-seed balanced pairs. The 2,100 completed-command-buffer samples per side measured GPU p50/p95/p99/peak of 1.535188/1.571794/1.611076/1.797125 ms on the parent and 0.851250/0.881877/0.902752/1.121250 ms on the candidate. Paired median speedup was 44.686% with 95% CI 44.511%..45.041% and 15/15 pair wins.

Both sides retained exactly one raster, one seed, nine JFA, and one compositor pass per measured frame with zero steady resource creation. Four-entry Metal target ownership fell from 69,599,232 to 19,136,512 allocated bytes. Aggregate frame p50/p95/p99 fell from 3.139000/3.544500/3.633500 to 1.605458/1.657292/3.169583 ms, while encode p50/p95/p99 fell from 0.121458/0.479000/0.549083 to 0.089709/0.136333/0.151791 ms. Metal exposed no pass-stage timestamp samples, so completed-command-buffer GPU duration is authoritative; logical JFA traffic derives from the exact formats and shader access counts rather than an unavailable hardware bandwidth counter.

An independent compositor-only guardrail collected another 2,100 submissions per side because C32 makes this the common cached path. Each submission encoded 512 identical real compositor passes so fixed GPU preemption could not dominate the roughly 0.07 ms single-draw duration. Parent/candidate GPU p50/p95/p99/peak were 19.922000/26.825356/27.136313/27.389417 and 19.633042/26.995108/27.251474/27.505500 ms. The fixed 5% no-material-regression policy accepted changes of -1.451%/+0.633%/+0.424%/+0.424%; both sides encoded exactly 71,680 measured compositor passes per process with zero steady resource creation.

Correctness proof compares every decoded field pixel at 256, 512, 1024, 2048, 257x509, 2048x257, and 511x1024 against CPU references. The asymmetric multi-seed/seam fixture also matches the complete CPU jump schedule exactly. Ten unchanged-parent/candidate BGRA captures cover Beauty, SeamMask, CityIdMask, and NeighborhoodIdMask; glow on/off; zero/default/wide polish; and multiple background alphas. Every capture is byte-identical. The coordinate selector accepts dimensions through 65,535 (maximum coordinate 65,534) and selects the compiled wide fallback at 65,536.

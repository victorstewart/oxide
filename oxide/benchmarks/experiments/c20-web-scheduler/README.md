# C20 Web scheduler evidence

C20 replaces synchronous resize/redraw rendering and the unconditional two-frame settle policy with a single pending-RAF contract. Canvas CSS geometry, backing dimensions, DPR, and pointer offsets are cached. ResizeObserver, captured scroll, window resize/DPR, and style/class/child-list mutation observation invalidate the cache; a dirty cache is refreshed once before the next frame or pointer-coordinate conversion. Static scenes stop after their requested frame, while animated scenes continue only through `Router::wants_next_frame`.

The performance hypothesis is that the old host performs one layout-affecting canvas query on every submitted frame, another bounding-rect query on every pointer sample, two submissions per ordinary invalidation, and one synchronous submission per resize or `oxide-redraw` event. The affected phases are browser event delivery, style/layout, RAF scheduling, Rust scene update/draw, WebGPU submit, and idle. Expected movement is 100-event resize/redraw storms from 101 submissions to one, click/key from two submissions to one, pointer layout reads from event-plus-frame cardinality to zero while geometry is clean, and static idle to zero callbacks/submissions after the requested frame. The main risks are stale pointer coordinates, missed CSS/DPR resize, animation starvation, observer feedback loops, timestamp drift, or an event-to-visible regression.

The real-Chrome adapter drives 100 settled click groups, 100 settled key groups, one second/240 samples of 240 Hz pointer input with RAF deadline sampling, 100-event resize and redraw storms, a CSS width mutation, and static idle. It retains every input event-to-visible sample before deriving per-process distributions, and records submissions/event, RAF callbacks/event, cached/forced metric reads, missed frames, cached geometry, and exact source/binary identities. Large per-pair process artifacts, the instrumentation-only parent counters, traces, paired inputs/reports, and captures remain in the ignored `raw/` directory. Official `latest.*` browser baseline promotion remains deferred to C62.

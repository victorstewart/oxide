# C22 retained node chunk evidence

C22 replaces flattened per-node and whole-surface retained draw lists with immutable node chunks and persistent ordered sequence references. Clean snapshots reuse the complete immutable snapshot when sequence identities, properties, and damage match. A dirty leaf replaces one chunk and path-copies only bounded composition blocks along its ancestor chain; unchanged text, image, and sibling chunks retain pointer identity.

The permanent architecture cases cover depths 16 and 32 with 1,000 label nodes, 500 image nodes, and independent retained text/image chunks. They report rebuilt/reused chunks and sequences, commands traversed/copied, geometry bytes copied, retained chunk and sequence bytes, and flat fallback use. Focused tests compare explicit compatibility flattening with direct output, including animated transforms/opacity, nested clip translation/intersection, layer translation, resource generations, namespace changes, ordering, and non-finite composed origins.

The locally ignored `raw/` directory is the stable evidence location for exact parent/candidate identities, the temporary probe patch and hash, randomized pair order, individual samples, statistical reports, binary hashes, allocation results, and final acceptance checks. The retained representation trades C21's single flattened surface allocation for independently replaceable immutable node metadata: on the depth-32 dirty workload, render-time allocation calls move from 14 to 158 and allocated bytes from 2,864 to 28,752, while scoped retained-cache live bytes move from 872 to 862,936. These are explicit costs of C22's fine-grained invalidation and make C23's byte budgets, admission, eviction, and churn resistance mandatory rather than optional. Aggregate workspace baseline refresh remains deferred to C62.

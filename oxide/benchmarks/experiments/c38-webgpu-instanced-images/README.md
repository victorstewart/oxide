# C38 WebGPU compact image-instance evidence

C38 replaces each ordinary WebGPU image's four generic packed vertices and six indices with one 36-byte destination/UV/alpha instance over a persistent four-vertex, six-index unit quad. Adjacent instances share one ordered draw only while texture, texture format, target, and clip remain compatible. The texture slot therefore remains the ordered draw binding, while prepared transforms and opacity remain in the existing property record. Image meshes, nine-slices, and SDF glyphs keep their generic indexed paths.

The isolated real-Chrome matrix runs 100 and 1,000 images in same-texture and mixed-texture sequences. Every row uses the same fractional destinations, four normalized source crops, four opacity levels, and outer clip in parent and candidate builds. The mixed rows switch texture every eight commands to verify that batching never reorders bindings. Parent and candidate packages use the same host-only adapter, native arm64 Chrome/Metal, queue-completed samples, and in-app WebGPU timestamp queries.

The candidate uploads 3,600/36,000 bytes for 100/1,000 images instead of the parent's 9,200/92,000 bytes, a 60.9% reduction. It preserves one draw/bind for same-texture rows and 13/125 draws and binds for mixed 100/1,000 rows. The three-pair pre-freeze population showed CPU p50 reductions of 3.7–22.3% and p95 reductions of 4.4–22.9%. Direct GPU p50 changed by +0.9–2.7% and p95 by -0.4–2.7%, with every candidate median below 0.075 ms. C38 is accepted for the CPU/upload reduction with exact pixels and unchanged draw/bind counts; it does not claim a GPU win. The final exact-tree raw population is the acceptance source of truth.

The deterministic 8×8 image capture covers two RGBA textures, four crops, four opacity levels, fractional destination sizes, and clipping. Parent/candidate pixels match exactly at DPR 1, 2, and 3. The 1,200×800 prepared snapshot also matches exactly, proving the static quad through prepared direct/bundle ownership plus dynamic transform/opacity property records. Aggregate committed browser/workspace baseline promotion remains deferred to C62.

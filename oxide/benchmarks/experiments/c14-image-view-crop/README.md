# C14 cropped ImageView evidence

This directory records C14 correctness and performance evidence for replacing `ImageView`'s zero-inset `NineSlice` misuse with bounded, source-pixel-cropped `Image` commands.

The performance hypothesis is that cover grids currently force one texture-specific nine-slice lowering and an oversized shaded quad per view. Emitting the semantic image primitive should let Metal use its cross-texture argument-table batches, reduce inline parameter bytes and shaded pixels, and preserve one quad per image. The affected stages are ImageView authoring, draw-list lowering, and fragment coverage; target workloads are 100 and 1,000 unique 29x7 images in 24x12 cover cells. Expected movement is 100/1,000 nine-slices to zero, 0 to 100/1,000 cropped images, 100/1,000 instanced draw calls to 2/16, 8,800/88,000 total parameter bytes to 7,432/72,256, and 59,657/596,571 logical shaded pixels to 28,800/288,000. The correctness risk is changing filtering at fractional source boundaries or allowing content outside the view; exact clipped-parent parity and DPR 1/2/3 odd-source goldens cover contain, cover, stretch, zoom, pan, and alpha.

The locally ignored `raw/` directory retains balanced parent/candidate samples, plans, hashes, counters, and captures. Official `latest.*` baselines remain deferred to C62.

The first correct crop implementation redundantly routed its already-proven rectangles through the public generic image validator and was rejected: 15-pair 100/1,000-image authoring medians regressed 29.53%/35.16%, with 0/15 wins for both. The retained crate-private prevalidated emission still honors the effective clip stack and moves the same controls from 1.068540/1.091711/1.092602/1.092825 us to 1.054152/1.076571/1.081775/1.083076 us for 100 images and from 8.358329/8.433474/8.446217/8.449403 us to 8.229848/8.324634/8.330571/8.332055 us for 1,000. Both no-material-regression gates accept; paired medians improve 1.49%/1.39% with 12/15 and 13/15 wins.

Preliminary backend evidence on the same retained command shape accepted all performance gates. Metal encode paired medians improved 28.44% for 100 images and 39.73% for 1,000; direct command-buffer GPU medians improved 14.21% and 61.05%. Deterministic counters move from 100/1,000 zero-slice nine-slices to 100/1,000 source-cropped images, 100/1,000 to 2/16 instanced draw calls, 8,800/88,000 to 7,432/72,256 total parameter bytes, and 59,657/596,571 to 28,800/288,000 logical shaded pixels while quads remain 100/1,000.

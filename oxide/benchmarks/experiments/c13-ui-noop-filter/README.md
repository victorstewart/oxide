# C13 central no-op draw rejection evidence

This directory records C13 correctness and performance evidence for rejecting semantically empty drawing in `DrawListBuilder` before backend lowering.

The performance hypothesis is that invisible authoring output consumes command storage, traversal, parameter upload, batching, and GPU draw/instance work even though it cannot change pixels. Central intrinsic visibility checks plus effective-empty-clip suppression should reduce those counters in proportion to discarded commands. The affected phases are UI draw emission and renderer lowering; target workloads are 4,096 transparent-container draws and 4,096 zero-area draws, each mixed with 64 visible controls. Expected movement is 4,160 to 64 emitted/traversed instances with corresponding upload/draw reduction. The correctness risk is removing a command that affects clip, layer-cache, or backdrop ordering; structural clip/layer commands remain, and unit/property plus Metal golden coverage freezes the boundary.

Permanent CPU and Metal architecture cases cover transparent-container and zero-area stress. The locally ignored `raw/` directory records balanced parent/candidate timing, command, instance, upload, and pixel evidence. Official `latest.*` baselines remain unchanged until C62.

Both accepted Metal encode A/B runs used 15 balanced fresh-process pairs, 20 warmups, and 80 measured frames per side/process. Transparent-container p50/p95/p99/peak moved from 0.046875/0.060504/0.077142/0.144709 ms to 0.009583/0.019625/0.029755/0.068167 ms; the paired median improved 80.08%, 95% CI was [76.99%,80.38%], all 15 pairs won, and the performance gate accepted. Zero-area p50/p95/p99/peak moved from 0.043959/0.060970/0.076627/0.132625 ms to 0.009688/0.020210/0.029337/0.068292 ms; the paired median improved 78.47%, 95% CI was [76.23%,80.17%], all 15 pairs won, and the gate accepted.

Full-frame wall time was retained as a diagnostic rather than the primary metric because unrelated command-buffer completion spikes dominated its tail. The transparent control accepted after the permitted doubling, while the zero-area control still had a candidate p95 spike despite better p50, p99, and peak; that mixed control is recorded as rejected evidence rather than presented as a win. Backend encode is the exact affected lowering phase and passes every median, confidence, win, p95, p99, and peak gate.

For both workloads, early rejection reduced emitted and traversed commands and GPU primitive instances from 4,160 to 64, instanced draw calls from 49 to 1, and inline RRect parameter bytes from 266,248 to 4,104 per frame. Exact raw-versus-filtered Metal pixels match for transparent and zero-area commands. The mixed clip/layer/backdrop snapshot equals an explicit visible-only command stream and its reviewed golden, while preserving structural layer and clip commands.

# C35 WebGPU ID-mask field-packing evidence

C35 stores city-seed XY in `.xy` and seam-seed XY in `.zw` of two ping-ponged `RGBA16Uint` fields when the adapter exposes render-attachment, texture-binding, and copy-source usage for that format and both dimensions fit the reserved `0xFFFF` invalid-coordinate contract. The final compositor resolves city and neighborhood IDs from the authoritative R8 masks. The existing four-`RGBA16Float` field shaders, pipelines, bindings, readback, and accounting remain prebuilt as the exact capability or coordinate-overflow fallback.

The packed fields use 16 logical bytes per pixel across both ping-pong textures versus 32 for the four wide fields, an exact 2x field-storage reduction. Total city, neighborhood, and field target ownership falls from 34 to 18 bytes per pixel. Each packed JFA candidate fetches one eight-byte city/seam coordinate record instead of two eight-byte semantic records, while strict comparison order preserves deterministic ties.

The controlled 512-square forced-miss calibration used unchanged C34 parent `8873c6e8a2d2e1479311dc1a366ac0f3c858db4b`, candidate runtime tree `a0c9a05f71141e39628af5214c614570872326d1`, identical instrumentation patch `ca22f8aa761527fa556e9f88295bfa16ac2b526b99e950bd4d7cadc4a6a28bbe`, fresh native-arm64 Chrome processes, 64 RAF warmups, 140 measured RAF submissions, and fifteen fixed-seed balanced pairs. Paired median speedup was 29.443% with 95% CI 29.219%..30.013% and 15/15 pair wins.

Every measured submission retained exactly one raster, one seed, nine JFA, and one compositor pass with zero steady texture creation.

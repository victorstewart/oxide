# C26 dynamic-property-slot evidence

C26 keeps retained text and image geometry in node-local coordinates and patches animation through stable generation-checked transform and opacity slots. The performance hypothesis is that animated frames are dominated by rebuilding whole-surface geometry, copying immutable commands, and uploading unchanged vertices rather than by the two small values that actually change. The proof workload animates 300 nodes containing 200 text subtrees and 100 image subtrees.

`NodeTree` owns dense transform and opacity slots, composes nested affine transforms and opacity into renderer properties, retains dynamic clip transforms, and retires reused indices under a new generation. Hit testing and accessibility frames consume the same composed affine state. `Animator` replaces its per-frame `BTreeMap` and repeated finished-entry removal with generation-checked dense override slots and one-pass retention. `UiSurface` no longer routes nonempty animation overrides around retained snapshots.

Metal and WebGPU keep dynamic records in separate completion-protected property rings. Both backends preserve immutable prepared geometry while animation changes. Metal tracks the last record revision for every physical frame slot and coalesces sparse changed runs; the common 300-record animation epoch uses snapshot-prevalidated all-record metadata. WebGPU binds the property ring through frame-local dynamic offsets. Static WebGPU snapshots remain on C25's aggregate render-bundle path, while property-driven snapshots use ordered prepared draws so transforms, opacity, clips, and resources remain exact.

The permanent CPU rows are `cpu.architecture.animation.surface_300` and `cpu.authoring.animation.dynamic_properties_300`; the backend row is `gpu.architecture.animation.dynamic_properties_300`. Correctness tests cover nested translation/scale/rotation/opacity, clips, retained equivalence, hit testing, accessibility frames, interruption, completion, generation reuse, stale-ID rejection, property-ring completion safety, and the unchanged C25 static-bundle boundary.

Final evidence uses 15 balanced fresh-process parent/candidate pairs for the shared 300-node `UiSurface` case and 15 user-interactive-QoS pairs with 15,000 samples per side for each Metal encode, event-to-submit, and direct command-buffer GPU distribution after 128 warmup frames per process. Every candidate frame must report 300 prepared-cache hits, zero misses, zero command traversal, zero geometry copies/uploads, exactly 300 property records and 14,400 property bytes, and zero missed 120 Hz deadlines. The browser guard independently requires the same 300/0 cache result and zero immutable work in real Chrome WebGPU. Exact tree, binary, instrumentation, population, distribution, and decision identities live in the locally ignored `raw/` directory. Aggregate workspace and physical-device baseline promotion remains deferred to C62.

The first backend revision attempted sparse per-instance revision hashing even when all 600 input properties advanced under one animation epoch. Its 15-pair Metal population failed the p95 and p99 encode gates, so that implementation was rejected. The retained path prevalidates the uniform epoch once while constructing the immutable snapshot, uses a const-specialized all-record update, and preserves exact sparse caching for mixed or partially changing property sets.

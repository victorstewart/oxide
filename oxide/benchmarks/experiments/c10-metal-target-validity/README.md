# C10 Metal persistent-target validity evidence

This directory records the C10 correctness and performance evidence for guarding partial-damage `Load` actions with explicit persistent-target validity.

The performance hypothesis is that the bug is a target-lifetime/state-validity defect, not an encoding bottleneck: one retained validity bit, a compact policy fingerprint, and predictable branches in the full-frame direct path should prevent undefined target loads while leaving CPU encode latency statistically indistinguishable. The affected phase is Metal render-pass target/load selection; the target workload is normal full-frame direct presentation. Expected counter movement is exactly one `damage_forced_full_refreshes` event after each direct-present or resize invalidation, then zero on ordinary valid frames. The visual-equivalence risk is stale or uninitialized pixels outside the damage scissor if any invalid target reaches `MTLLoadActionLoad`.

The sequence regression renders full direct -> partial damage -> full -> resize -> partial with distinctive colors. It reads the synthetic present texture for the direct frame, compares both forced refreshes byte-for-byte with fresh full renders, and freezes all visible states as PNG goldens. Metal validation runs the same sequence. The locally ignored `raw/` directory records the balanced parent/candidate direct-frame timing evidence. Official `latest.*` baselines remain unchanged until C62.

The optimized A/B uses 15 balanced fresh-process pairs, 20 warmups and 100 measured frames per side/process, a reused 1,200 x 800 synthetic present texture, and 256 rounded rectangles. Across 1,500 samples per side, parent/candidate p50 was 0.014000/0.013833 ms, p95 was 0.021875/0.021250 ms, p99 was 0.026551/0.024751 ms, and peak was 0.042833/0.033208 ms. The paired median change was +0.59% candidate time with a 95% bootstrap interval spanning -8.35% to +6.34%; the shared no-material-regression policy accepted the result with no tail or peak regression. The state machine therefore has no measurable full-frame direct hot-path cost at this resolution.

`MTL_DEBUG_LAYER=1 cargo test -p oxide-renderer-metal --tests --features snapshot-tests --no-fail-fast` passed 71 tests, and `MTL_DEBUG_LAYER=1 cargo test -p oxide-snapshot-runner --no-fail-fast` passed 20 tests with Metal API Validation enabled. `cargo run -p xtask -- experiments check` accepted all 94 decided manifest entries.

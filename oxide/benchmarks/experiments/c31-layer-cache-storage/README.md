# C31 bounded layer-cache storage evidence

C31 gives retained layer textures a hard byte budget on Metal and WebGPU, protects the current frame from eviction, recycles exact-compatible textures through a bounded pool, evicts least-recently-used absent layers, ages cold pool entries, and purges storage explicitly for memory pressure, device loss, scale changes, and application requests. Metal admission uses `heapTextureSizeAndAlign` and reports actual `allocatedSize`; WebGPU reports exact logical texture bytes because the browser API does not expose driver allocation size.

If a complete current-frame layer set exceeds the budget, the renderer uses the exact inline or flat path instead of admitting only part of the set. Ordinary resize preserves compatible storage, scale changes purge it, and the iOS memory-warning path now reaches the Metal owner. Telemetry separates resident GPU bytes, pooled GPU bytes, logical CPU metadata bytes, oldest-use frame, reuse, eviction, recreation, purge count, purge reason, and budget violations.

The permanent `gpu.architecture.layers.*` rows run with a 16 MiB benchmark budget and persist the new counters. The navigation probe uses 100 new 32×32 layers per frame, four warmups, 140 measured frames, fifteen balanced fresh-process pairs, and 2,100 completed-command-buffer timestamps per side. Against the unchanged C30 parent, peak layer residency fell from 51,110,400 to 422,400 bytes (99.174%), recurring layer texture creations fell from 210,000 to zero, frame p50/p95/p99 improved 6.434%/8.468%/20.746%, and encode p50/p95/p99 improved 37.297%/43.703%/60.890%.

Direct GPU time is a no-material-regression guardrail for this storage/CPU optimization. GPU p50/p95/p99/peak moved +0.101%/+0.103%/+0.286%/+1.699%, all within the fixed 5% tail guardrail, so the immutable paired population is accepted under that declared policy. The generic speedup policy report is retained as rejected because it requires a 5% GPU improvement and 80% pair wins, neither of which is C31's hypothesis; no samples or thresholds were changed between the two analyses.

Native-arm64 Chrome exercised 100-card WebGPU navigation under a 6 MiB budget. The candidate held peak residency to 4,608,000 bytes, reused textures 6,000 times, performed zero steady-state texture creations, recorded zero budget violations, and returned resident and pool bytes to zero after both memory-pressure and simulated device-loss purges. Its 60-frame distribution was p50 0.970 ms, p95 1.840 ms, p99/peak 2.490 ms. Metal validation snapshots and focused runtime contracts preserve exact pixels, early-frame age boundaries, resize/scale behavior, reentry, and purge semantics.

The locally ignored `raw/` directory retains the instrumentation patch, production-code tree and binary identities, balanced order, every CPU/GPU timestamp, browser health counters, workload counters, accepted guardrail report, and the intentionally rejected GPU-speedup-policy report. Aggregate workspace and physical-device baseline promotion remains deferred to C62.

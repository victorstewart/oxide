# C17 Metal effect-target evidence

C17 removes the unused full-resolution blur temporary and makes retained Metal effect resources follow the declared render-pass plan. Direct UI owns no effect targets; a zero-blur backdrop owns only its full-resolution prepass; ordinary blur adds half/quarter plus one quarter ping-pong target; strong visual blur uses half/quarter plus the two declared eighth-resolution targets. Compatible targets survive warm frames. Resize discards incompatible targets, and the production iOS memory-warning path purges effect and bloom targets before requesting a replacement frame.

The performance hypothesis is that eager allocation inflates first-effect CPU work and private texture residency. The affected stage is Metal effect pass preparation and resource lifetime. The target workloads are fresh-renderer direct, prepass-only, quarter-blur, and eighth-blur frames at 1200x800. Expected movement is parent seven-target residency for every effect to candidate 1/4/5 target creation, zero warm growth, and no direct-path target ownership. The correctness risk is a missing attachment, stale-size reuse, lost memory-pressure rebuild, or changed filtering/pixels.

The exploratory exact-tree A/B used the shared paired runner, identical instrumentation, fresh processes, balanced order, 20 pairs, and 100 cold first-use encode samples per side/process. Prepass improved 38.738% with 95% CI 36.973..39.761, quarter improved 12.732% with CI 9.866..15.006, and eighth improved 8.960% with CI 7.500..10.044; every candidate won 20/20 pairs. The unchanged direct encode control passed after the permitted doubling to 40 pairs: p50 0.018458 -> 0.018458 ms, paired +0.226%, CI -0.977..2.837, 21/40 wins, with improved p95/p99/peak.

At 1200x800, parent effect residency was 10,305,536 bytes for prepass, quarter, and eighth workloads. Candidate residency is 4,112,384 bytes for prepass (-60.09%), 5,963,776 for quarter (-42.13%), and 5,849,088 for eighth (-43.24%). Direct UI remains zero effect/bloom bytes on both sides. Candidate first-use creation is now honestly counted as 0/1/4/5, and a second warm quarter frame creates zero resources.

Four committed Metal goldens cover direct, prepass, quarter, and eighth output. The locally ignored `raw/` directory retains plans, all raw samples, identity/hardware hashes, exploratory/final paired reports, the rejected whole-frame wall control, and exact parent validation. Official `latest.*` promotion remains deferred to C62.
